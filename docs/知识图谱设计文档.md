# 知识图谱设计文档

## 1. 本体设计

### 1.1 实体类型
| 实体类型 | 说明 | 示例 |
|---------|------|------|
| 设备类 | 电力设备实体 | 变压器/断路器/隔离开关 |
| 缺陷类 | 缺陷类型实体 | 油液泄漏/锈蚀/裂纹 |
| 标准类 | 标准导则实体 | 运行规程/检修导则 |
| 案例类 | 历史案例实体 | 故障案例/检修案例 |
| 专家经验类 | 专家知识实体 | 诊断经验/检修经验 |

### 1.2 关系类型
| 关系类型 | 源实体 | 目标实体 | 说明 |
|---------|--------|---------|------|
| 属于 | 设备 | 设备类型 | 设备分类关系 |
| 导致 | 缺陷 | 故障 | 缺陷导致故障 |
| 参考 | 案例 | 标准 | 案例参考标准 |
| 类似 | 案例 | 案例 | 案例相似关系 |
| 位于 | 设备 | 位置 | 设备安装位置 |
| 连接 | 设备 | 设备 | 设备连接关系 |

## 2. 知识抽取

### 2.1 设备台账抽取
- 来源：PMS 系统/设备管理系统
- 方法：结构化数据导入
- 字段：设备 ID/名称/类型/参数/位置

### 2.2 缺陷知识抽取
- 来源：缺陷管理系统
- 方法：规则抽取 + NLP
- 字段：缺陷类型/等级/原因/处理方案

### 2.3 标准导则抽取
- 来源：标准文档（PDF/Word）
- 方法：OCR+NLP+ 人工校对
- 字段：标准号/名称/条款/内容

### 2.4 案例知识抽取
- 来源：历史案例库
- 方法：NLP+ 人工标注
- 字段：案例描述/原因/处理/经验

### 2.5 专家经验抽取
- 来源：专家访谈/手册
- 方法：知识工程 + 人工整理
- 字段：经验描述/适用场景/注意事项

## 3. 知识存储

### 3.1 图数据库选型
- **Neo4j**：成熟度高/社区活跃/查询语言完善
- **Nebula Graph**：国产/分布式/高性能
- **选型**：Neo4j（社区版）

### 3.2 Schema 设计
```cypher
// 创建约束
CREATE CONSTRAINT device_id IF NOT EXISTS FOR (d:Device) REQUIRE d.id IS UNIQUE
CREATE CONSTRAINT defect_id IF NOT EXISTS FOR (d:Defect) REQUIRE d.id IS UNIQUE

// 创建索引
CREATE INDEX device_name IF NOT EXISTS FOR (d:Device) ON (d.name)
CREATE INDEX defect_type IF NOT EXISTS FOR (d:Defect) ON (d.type)
```

### 3.3 数据导入
- 工具：Neo4j Import Tool / Cypher LOAD CSV
- 批量导入：10 万节点/分钟
- 增量更新：定时任务 + 变更捕获

## 4. 知识检索

### 4.1 查询模板
```cypher
// 查询设备关联缺陷
MATCH (d:Device {id:$deviceId})-[:HAS_DEFECT]->(def:Defect)
RETURN def

// 查询相似案例
MATCH (c1:Case {id:$caseId})-[:SIMILAR_TO]->(c2:Case)
RETURN c2

// 查询推荐标准
MATCH (c:Case)-[:REFERENCE]->(s:Standard)
WHERE c.id = $caseId
RETURN s

// 查询设备健康度
MATCH (d:Device {id:$deviceId})
RETURN d.healthScore
```

### 4.2 智能问答
- 问题解析：NLP+ 意图识别
- 图谱查询：Cypher 生成
- 答案生成：模板 + NLG
