# 预测性维护算法说明

## 1. 算法概述

### 1.1 算法架构
```
设备数据 → 特征工程 → 健康度评估 → 趋势预测 → 故障预警 → 维护建议
```

### 1.2 核心算法
- **健康度评估**：多维度加权评分
- **趋势预测**：LSTM/Prophet 时间序列预测
- **故障预警**：阈值告警 + 异常检测
- **维护建议**：规则引擎 + 知识图谱

---

## 2. 健康度评估模型

### 2.1 评估维度
| 维度 | 权重 | 指标 |
|------|------|------|
| 运行状态 | 30% | 负载率/温度/振动 |
| 缺陷历史 | 25% | 缺陷数量/缺陷等级/重复缺陷 |
| 维护记录 | 20% | 维护频次/维护质量 |
| 设备年限 | 15% | 投运年限/设计寿命 |
| 环境影响 | 10% | 温度/湿度/污染等级 |

### 2.2 评分公式
```
健康度 = Σ(维度得分 × 权重)

维度得分 = Σ(指标得分 × 指标权重)

指标得分 = f(实际值，标准值，阈值)
```

### 2.3 健康等级
| 等级 | 分数 | 说明 | 处置建议 |
|------|------|------|---------|
| 优秀 | 90-100 | 设备状态良好 | 正常巡检 |
| 良好 | 80-89 | 设备状态正常 | 加强监视 |
| 注意 | 70-79 | 设备存在隐患 | 缩短巡检周期 |
| 异常 | 60-69 | 设备状态异常 | 安排检修 |
| 严重 | <60 | 设备严重异常 | 立即停电检修 |

---

## 3. 趋势预测算法

### 3.1 LSTM 预测模型

**网络结构**：
- 输入层：历史 30 天数据（温度/负载/缺陷）
- LSTM 层：2 层，每层 64 个神经元
- Dropout：0.2
- 全连接层：1 个神经元（预测值）
- 输出层：未来 7 天预测值

**训练参数**：
- 损失函数：MSE
- 优化器：Adam
- 学习率：0.001
- Batch Size：32
- Epochs：100

### 3.2 Prophet 预测模型

**模型组件**：
- 趋势项：分段线性增长
- 季节项：周季节 + 年季节
- 节假日项：特殊日期影响
- 回归项：外部变量（天气/负载）

**预测输出**：
- 未来 7/15/30 天趋势预测
- 置信区间（80%/95%）
- 异常点检测

## 4. 故障预警规则

### 4.1 阈值告警
| 告警类型 | 触发条件 | 告警级别 |
|---------|---------|---------|
| 温度过高 | 温度>注意值 | 一般 |
| 温度严重 | 温度>警示值 | 严重 |
| 负载过重 | 负载率>90% | 一般 |
| 健康度下降 | 健康度<70 | 注意 |
| 健康度严重 | 健康度<60 | 严重 |

### 4.2 趋势告警
- 温度持续上升（3 天连续上升）
- 健康度持续下降（7 天下降>10 分）
- 缺陷频发（7 天内 3 次同类缺陷）

### 4.3 关联告警
- 多设备同时异常
- 同一设备多参数异常
- 环境异常导致设备异常

## 5. 维护建议生成

### 5.1 规则引擎
```
IF 健康度 < 60 THEN 立即停电检修
IF 健康度 < 70 AND 温度>注意值 THEN 安排检修（7 天内）
IF 健康度 < 80 AND 缺陷数量>3 THEN 加强监视（3 天 1 次）
IF 预测温度>警示值 THEN 提前干预（3 天内）
```

### 5.2 知识图谱推荐
- 相似案例推荐（历史类似故障）
- 标准导则推荐（检修规程）
- 备品备件推荐（所需材料）
- 检修队伍推荐（专业班组）
